# Day2 单机并发缓存
1、支持并发：sync.Mutex 互斥锁

2、目录结构
geecache/
    |--lru/
        |--lru.go  // lru 缓存淘汰策略
    |--byteview.go // 缓存值的抽象与封装
    |--cache.go    // 并发控制
    |--geecache.go // 负责与外部交互，控制缓存存储和获取的主流程

3、 Group 是 GeeCache 最核心的数据结构，负责与用户的交互，并且控制缓存值存储和获取的流程。
缓存流程：
    检查是否被缓存
    是否应当从远程节点获取
    调用回调函数，获取值并添加到缓存
